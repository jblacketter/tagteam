### Reviewer

**Action: REQUEST_CHANGES**

Good progress, but there are blocking implementation issues to resolve before approval:

**Blocking 1: Structured rounds viewer is not actually wired**
- `app.js` still loads raw markdown via `loadCycleDoc()` from cycle selection flow.
- `loadRounds()` is defined but never called, and `#rounds-content` remains hidden.
- Success criterion "web dashboard displays structured rounds (not raw markdown)" is not met.

**Blocking 2: /api/rounds response schema and frontend renderer are mismatched**
- `server.py` returns parser-shaped rounds (`round`, `lead_summary`, `reviewer_summary`, etc.) plus `html`.
- `app.js` `renderRounds()` expects a different schema (`round.number`, `round.turns`, `round.result`), so rendered structured content is incorrect/empty.
- Align frontend rendering with backend contract (or consume `data.html` directly).

**Blocking 3: Parser action extraction regressed compatibility with existing cycle docs**
- `ai_handoff/parser.py` action regex matches `**Action: X**` but does not match existing docs that use `**Action:** X`.
- This drops `lead_action`/`action` for legacy cycle docs and breaks status/action display continuity.
- Update parser to support both action formats and add regression tests for legacy syntax.

**Non-blocking**
- `tui/app.py` stale recovery path clears `_stale` flag but does not re-render status bar when state is unchanged, so `[STALE]` can remain visible until a later state change.

## Round 2

### Lead

**Action: SUBMIT_REVISED_PLAN**

Addressed all 3 blocking issues and the non-blocking issue:

**Blocking 1 — Structured rounds viewer wired:** `loadCycleDoc()` in `app.js` now calls `loadRounds(filename)` alongside the raw markdown load, so structured rounds are displayed when any cycle doc is selected.

**Blocking 2 — Frontend/backend schema aligned:** `renderRounds()` in `app.js` now uses the pre-rendered `data.html` from the backend (generated by `format_rounds_html()` in `parser.py`) instead of trying to consume a mismatched schema. The `#rounds-content` div is unhidden when data is available.

**Blocking 3 — Legacy action format support:** Parser regex in `parser.py` changed from `\*\*Action:\s*(\w+)` to `\*\*Action:(?:\*\*)?\s*(\w+)`, which matches both `**Action: VALUE**` (new format) and `**Action:** VALUE` (legacy format). Added 2 regression tests: `test_legacy_action_format` (full round extraction) and `test_skips_legacy_action_line` (summary extraction).

**Non-blocking — Stale recovery re-render:** `_poll_state()` in `tui/app.py` now calls `status_bar.update_state()` when recovering from stale, ensuring the `[STALE]` badge is cleared immediately even if state is unchanged.

**Test results:** 98 passed, 1 failed (pre-existing `test_stops_after_finding_both`).
