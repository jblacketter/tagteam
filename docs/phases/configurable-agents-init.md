## Summary
**What:** Create an interactive init command that configures two AI agents with roles (lead/reviewer), generates `ai-handoff.yaml`, and updates skills/templates/docs to use role-based language.

**Why:** New users need an easy onboarding experience. Currently the framework hardcodes Claude/Codex, limiting adoption for users with different AI combinations or role preferences.

**Depends on:** None (foundational feature)

## Scope

### In Scope
- Interactive `init` command prompting for 2 agents and their roles
- CLI entrypoint via `ai_handoff/__main__.py`
- `ai-handoff.yaml` config file in project root
- Safe overwrite flow when config already exists
- All skills, templates, and docs updated to use role-based language
- Getting started instructions with identity hint
- Re-runnable init to swap roles

### Out of Scope
- More than 2 agents (future enhancement)
- Multiple leads/reviewers with specialties
- Model pattern matching for self-identification (AIs know who they are)
- Drift detection and merge strategies
- Windows-specific sync commands
- Migration from old setups (separate phase)
- Template variable substitution (skills read config directly)

## Technical Approach

### 1. Config Schema (Simplified)

```yaml
# ai-handoff.yaml
agents:
  lead:
    name: claude
  reviewer:
    name: codex
```

Simple, flat structure. No complexity with arrays, primary flags, or model patterns.

### 2. CLI Entrypoint

Create `ai_handoff/__main__.py` as the module entrypoint:

```python
"""CLI entrypoint for python -m ai_handoff"""
import sys
from ai_handoff.cli import main

if __name__ == "__main__":
    sys.exit(main())
```

Create `ai_handoff/cli.py` with subcommand dispatcher:

```python
def main():
    if len(sys.argv) < 2:
        print("Usage: python -m ai_handoff <command>")
        print("Commands: init, setup")
        return 1

    command = sys.argv[1]
    if command == "init":
        return init_command()
    elif command == "setup":
        return setup_command()
    else:
        print(f"Unknown command: {command}")
        return 1
```

### 3. Init Command Flow

```
$ python -m ai_handoff init

AI Handoff Setup
================
This framework coordinates work between two AI agents.

Agent 1 name: claude
Agent 1 role (lead/reviewer): lead

Agent 2 name: codex
Agent 2 role (lead/reviewer): reviewer

✓ Created ai-handoff.yaml
✓ Ready to go!

Getting Started:
  Tell your AI: "I am [agent name]. Read ai-handoff.yaml to confirm my role,
  then read .claude/skills/ to understand the workflow."
```

**Safe Overwrite Flow:**
```
$ python -m ai_handoff init

ai-handoff.yaml already exists with:
  Lead: claude
  Reviewer: codex

Overwrite? (y/n):
```

**Validation:**
- Enforce exactly one lead and one reviewer
- If user enters same role twice, re-prompt: "You need one lead and one reviewer. Please re-enter Agent 2 role:"

### 4. Skill Updates (Runtime Config Reading)

Add a preamble to each skill:

```markdown
> **Before using this skill**:
> 1. Read `ai-handoff.yaml` in the project root
> 2. Identify which agent you are (lead or reviewer)
> 3. Follow the instructions for your role below
```

Replace hardcoded "Claude" and "Codex" with role-based language:
- "Claude" → "Lead" or "the lead agent"
- "Codex" → "Reviewer" or "the reviewer agent"

## Risks
- **Risk**: AIs might not reliably read the config file before acting
  - **Mitigation**: Bold preamble in every skill, identity hint in getting started prompt

- **Risk**: Editing ai-handoff.yaml manually could break format
  - **Mitigation**: Keep schema extremely simple (just 4 lines of YAML)

- **Risk**: CLI entrypoint not discoverable
  - **Mitigation**: Document in README, consider adding console_scripts entrypoint in pyproject.toml
